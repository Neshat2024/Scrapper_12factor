I. CODEBASE

Una base di codice tracciata nel controllo di revisione, molti distribuzioni

Un'app di dodici fattori viene sempre monitorata in un sistema di controllo della versione, comeIdiota,Mercuriale, OSovversione. Una copia del database di tracciamento della revisione è nota come arepository di codice, spesso abbreviato aCodice Repoo soloRepo.

UNA UNCODEBASEè un singolo repository (in un sistema di controllo di revisione centralizzato come la sovversione) o qualsiasi insieme di repository che condividono un commit radicale (in un sistema di controllo di revisione decentralizzato come Git).

Esiste sempre una correlazione individuale tra la base di codice e l'app:

  • Se ci sono più basi di codice, non è un'app: è un sistema distribuito. Ogni componente in un sistema distribuito è un'app e ciascuno può essere conforme individuale a dodici fattori.
  • Più app che condividono lo stesso codice sono una violazione di dodici fattori. La soluzione qui è fare il fatto di fare il codice condiviso in librerie che possono essere incluse attraversoDipendency Manager.

C'è solo una base di codice per app, ma ci saranno molti distribuzioni dell'app. UNdistribuireè un'istanza in esecuzione dell'app. Questo è in genere un sito di produzione e uno o più siti di stadiazione. Inoltre, ogni sviluppatore ha una copia dell'app in esecuzione nel proprio ambiente di sviluppo locale, ognuna delle quali si qualifica anche come distribuzione.

La base di codice è la stessa in tutte le distribuzioni, sebbene diverse versioni possano essere attive in ciascuna distribuzione. Ad esempio, uno sviluppatore ha alcuni commit non ancora distribuiti alla messa in scena; La messa in scena ha alcuni impegni non ancora dispiegati in produzione. Ma condividono tutti la stessa base di codice, rendendoli così identificabili come distribuzioni diverse della stessa app.

VII. Collegamento delle porte

Esporta i servizi tramite l'associazione delle porte

Le app Web vengono talvolta eseguite all'interno di un contenitore di server Web. Ad esempio, le app PHP potrebbero essere eseguite come modulo all'internoApache HTTPDoppure le app Java potrebbero essere eseguite all'internoGatto.

L'app a dodici fattori è completamente autonomae non si basa sull'inserimento in fase di esecuzione di un server Web nell'ambiente di esecuzione per creare un servizio con connessione Web. L'applicazione webesporta HTTP come servizio eseguendo l'associazione a una portae ascoltando le richieste in arrivo su quella porta.

In un ambiente di sviluppo locale, lo sviluppatore visita un URL di servizio comehttp://localhost:5000/per accedere al servizio esportato dalla loro app. Nella distribuzione, un livello di routing gestisce le richieste di routing da un nome host pubblico ai processi Web associati alla porta.

Questa operazione viene in genere implementata utilizzandoDichiarazione di dipendenzaper aggiungere una libreria del server Web all'app, ad esempioTornadoper Python,Sottileper Rubino, oMoloper Java e altri linguaggi basati su JVM. Questo avviene interamente inSpazio utente, ovvero all'interno del codice dell'app. Il contratto con l'ambiente di esecuzione è vincolante a una porta per soddisfare le richieste.

HTTP non è l'unico servizio che può essere esportato tramite port binding. Quasi tutti i tipi di software server possono essere eseguiti tramite un processo che si lega a una porta e attende le richieste in arrivo. Ecco alcuni esempi:ejabberd(parlandoXMPP), eRedis(Parlando ilProtocollo Redis).

Si noti inoltre che l'approccio di associazione delle porte significa che un'app può diventare ilServizio di supportoper un'altra app, fornendo l'URL all'app di supporto come handle di risorsa nel fileconfigurazioneper l'app di consumo.
